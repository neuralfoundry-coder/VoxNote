### TC-SRV-001-01: Google OAuth2 OIDC 로그인

| 항목 | 내용 |
|------|------|
| **테스트 ID** | TC-SRV-001-01 |
| **요구사항 ID** | FR-SRV-001 |
| **우선순위** | P1 |
| **테스트 유형** | 통합 |
| **사전 조건** | - Google OAuth2 클라이언트 ID/Secret 설정 완료<br>- OIDC Discovery URL 접근 가능<br>- 테스트용 Google 계정 준비 |

**테스트 절차:**

1. 클라이언트에서 Google 로그인 버튼을 클릭한다.
2. Google OAuth2 인증 페이지로 리다이렉트되는지 확인한다.
3. 테스트 Google 계정으로 인증을 완료한다.
4. 콜백 URL로 authorization code가 전달되는지 확인한다.
5. 서버에서 code를 token으로 교환하는지 확인한다.
6. OIDC id_token에서 사용자 정보가 추출되는지 확인한다.

**기대 결과:**
- Google OAuth2 인증 흐름이 정상 수행된다.
- authorization code가 access_token/id_token으로 교환된다.
- 사용자 이메일, 이름 등 프로필 정보가 추출된다.
- 신규 사용자의 경우 계정이 자동 생성된다.

**테스트 절차:**

1. 만료된 access_token으로 API 요청을 시도한다.
2. 401 응답을 수신한다.
3. refresh_token으로 토큰 갱신 엔드포인트를 호출한다.
4. 새로운 access_token이 발급되는지 확인한다.

- refresh_token으로 새 access_token이 발급된다.
- 새 access_token이 유효한 RS256 서명을 가진다.
- 이전 refresh_token이 무효화되고 새 refresh_token이 발급된다(rotation).
- 무효화된 refresh_token으로 갱신 시도 시 거부된다.
